# 系统说明
## 一、系统全景概述
### 1、产品定义
本产品是覆盖“医院心脏康复中心-社区康复站-患者家庭”三级场景的心脏康复管理平台。该系统旨在为冠心病、冠状动脉介入治疗术后、慢性心力衰竭稳定期等心血管疾病患者或有其他慢性病患者提供连续、规范且个性化的康复管理服务。核心功能在于整合并数字化心脏康复全流程：医院端医生可基于系统完成患者的专业医学评估(如心肺运动试验分析)，制定结构化的五大处方(运动、营养、心理、戒烟/限酒、药物)；社区端医护人员可接收并执行方案，组织监督线下康复训练并记录数据；患者则通过移动应用接收每日任务、上传居家生理数据(如通过蓝牙连接血压计、心电设备)、学习健康教育知识，并与医护团队保持沟通。

系统通过统一的数据中心实现三端信息的实时同步与安全交互，并内嵌智能预警算法对患者异常数据进行主动监测与提醒，从而构建一个以评估为基础、处方为核心、执行与监控为闭环、数据为驱动的远程康复管理工具，最终提升患者依从性、改善康复结局，并支持医疗机构开展标准化、规模化的心脏康复服务，解决传统康复脱落率高、依从性差、难以长期管理的痛点。

### 2、系统组成及目标用户
| 系统 | 目标用户 | 主要用途 |
| --- | --- | --- |
| 医院端(上级医院) | 康复医师 | 用于患者建档、全面评估(如心肺运动试验、量表评估)、制定个性化康复处方(运动、营养、心理、用药、戒烟)、监控全院/社区患者进展、进行远程指导与风险处置。 |
| 社区端(基层医院) | 社区医生或康复师 | 用于接收并执行医院下达的康复方案、组织患者进行本地化康复训练、记录社区训练数据、发现异常并向上转诊。 |
| 家庭端(患者终端) | 患者 | 通过移动App、可穿戴设备等，接收并执行每日康复任务、上传居家监测数据(心率、血压、体重、症状)、接受健康教育、训练评估、进行咨询或紧急报警。 |

## 三、功能模块描述
### 1、医院端
| 模块大类 | 功能模块 | 功能描述 |
| --- | --- | --- |
| 患者管理 | 档案管理 | 支持新建、编辑、查询患者基本信息，建立个人档案，包含基础身体指标(如身高、体重、心率、血压等)、既往病史、药物史、家族史、检验检查指标(如生化、血常规)并可归档 |
|  | 分层管理 | 根据患者疾病诊断(如心梗、心衰)、心功能分级、手术类型等进行标签化分组，并进行患者分流(转到专科门诊、社区医院或居家) |
|  | 授权管理 | 管理患者与医院/医生的服务协议，设置数据查看与共享权限(如对社区医生开放部分数据) |
| 评估管理 | 量表评估 | 集成标准化评估表(如SF-36生活质量量表、PHQ-9抑郁量表)，支持在线填写、自动评分与历史对比 |
|  | 器械评估 | 支持心肺运动试验(CPET)、体成分、三维步态等设备的数据接口导入或手动录入关键指标，并自动生成评估报告 |
|  | 自定义表单 | 支持自定义创建评估量表 |
|  | 风险评估 | 内置风险评估模型，根据患者的量表和器械评估数据自动计算风险等级，为处方制定提供依据 |
| 康复方案制定 | 方案编辑 | 提供标准化、可自定义的处方模板，包含运动治疗(类型、强度、频率、时长)、营养指导、心理干预、戒烟建议、药物调整建议五大处方模块 |
|  | 方案分发 | 可将制定好的方案同步至对应的医院门诊部、社区和患者家庭端APP，并附带执行说明 |
|  | 方案版本管理 | 记录方案每次调整的历史版本，方便追踪康复计划变更轨迹，并自动提醒社区和患者更新 |
|  | 运动器械管理 |  |
| 方案执行 | 康复训练记录 | 同步到医院门诊部的方案，并支持记录患者的训练情况 |
| 方案执行监控 | 全景数据监控面板 | 以仪表盘形式集中展示执行方案的患者实时状态，如心率、血压、预警信息、任务完成率等，支持按风险等级排序 |
|  | 智能预警设置 | 系统根据预设安全阈值(如心率超过上限)自动触发预警，并通过弹窗、短信等方式通知负责医生 |
|  | 依从性管理 | 自动统计患者处方执行完成率、数据上报及时率，标记依从性差的患者 |
| 随访管理 | 随访计划管理 | 支持创建标准化或个性化的随访计划，系统自动推送随访任务(如电话、问卷、复查提醒)给医护人员 |
|  | 医患沟通 | 提供安全的即时通讯工具，支持医患之间的文字、图片、语音沟通，所有记录留存归档 |
| 沟通协同 | 消息中心 | 1、接收来自社区端的系统通知(如上报转诊)、预警提醒，并支持与社区端医生进行安全的点对点消息沟通<br>2、接收患者端提交的运动完成情况和预警通知 |
| 数据统计 | 康复效果统计分析 | 自动生成个体或群体的康复效果对比报告(如运动耐力前后对比、危险因素控制率、指标变化趋势)，支持图表化展示 |
|  | 数据导出 | 支持根据研究需求，对脱敏后的患者数据进行筛选、打包和导出，服务于临床研究 |
|  | 指标看板 | 为科室医生提供业务看板，统计在管患者数、康复完成率、复诊率、患者满意度等管理指标 |
| 系统管理 | 权限管理 | 精细化的角色权限控制(如主任、医生、治疗师、护士)，不同角色看到的数据和操作权限不同 |
|  | 社区机构管理 | 管理下级协作的社区康复站点信息，分配及查看各社区的患者归属情况 |
|  | 系统接口管理 | 提供与医院HIS、LIS、PACS等系统的标准接口(如HL7、FHIR)，实现患者基本信息和检验检查结果的调取。 |
| 健康教育 | 科普管理 | 通过管理模块发布科普文章到患者端APP进行健康教育 |

### 2、社区端
| 模块大类 | 功能模块 | 功能描述 |
| --- | --- | --- |
| 患者管理 | 本社区患者列表 | 查看归属到本社区的所有患者的基本信息、康复阶段、近期依从性概况，支持按标签筛选 |
| 评估管理 | 简易评估 | 集成适合社区使用的简易评估表(如6分钟步行试验距离、Borg自觉疲劳量表)，支持快速录入并自动生成趋势图。 |
|  | 日常生理指标录入 | 支持手动或通过蓝牙连接设备(如血压计、体重秤)批量录入患者到站时的血压、心率、体重、血氧等基础数据。 |
|  | 异常事件上报 | 提供结构化表单，供患者或社区医生快速上报训练中的异常症状(如胸痛、头晕)，并立即触发预警流程 |
| 方案执行 | 执行方案看板 | 集中展示当日/本周所有待执行任务(如康复训练、随访)，按优先级和患者风险等级排序，支持一键进入。 |
|  | 康复方案接收查询 | 同步展示由医院端下达的个性化心脏康复处方，包含运动、教育、营养等具体项目、强度、频率及注意事项。 |
|  | 康复训练记录 | 支持在组织患者进行集体或个体训练时，一键记录训练项目的实际完成情况(如运动时长、达到的心率、疲劳程度) |
| 沟通协同 | 消息中心 | 1、接收来自医院端的系统通知(如处方调整、患者转入)、预警提醒，并支持与医院端医生进行安全的点对点消息沟通<br>2、接收患者端提交的运动完成情况和预警通知 |
|  | 快速转诊 | 发现患者情况超出社区处理能力，可通过标准化表单一键发起向医院端的转诊请求，并上传相关数据 |
| 健康教育 | 科普材料库 | 存放并播放由医院端统一制作或审核的心脏康复教育视频、图文资料，供社区组织患者集体学习或推送至家庭端 |
| 系统管理 | 权限管理 | 精细化的角色权限控制(如社区管理员、社区医生)，不同角色看到的数据和操作权限不同 |
|  | 医院管理 | 管理上级协作的医院信息，上报及查看各医院的患者情况 |

### 3、家庭端(患者终端)
| 模块大类 | 功能模块 | 功能描述 |
| --- | --- | --- |
| 个人中心 | 健康档案 | 基础健康档案(个人信息、既往史、药物史等)、医院器械检查指标(如有)、日常居家生理参数 |

| 评估管理 | 简易评估 | 集成适合居家使用的简易评估表(如6分钟步行试验距离、Borg自觉疲劳量表)，支持快速录入并自动生成趋势图。 |
|  | 分级管理(规则通过后台配置) | 1、引入风险分层引擎(AACVPR标准)自动分级(区分高中低危患者)，同时动态复评提醒(规则通过后台配置)<br>2、低危病人生成系统预设训练方案 |
| 方案执行 | 今日方案看板 | 首页清晰展示当日所有康复任务清单(如运动、服药、量血压、学习课程)，完成可打卡，形成日历视图 |
|  | 生理指标手动录入 | 支持手动录入患者居家的血压、心率、体重、血氧等基础数据。 |
|  | 运动跟练指导 | 提供预录制的标准化康复运动视频或图片(如热身、有氧、阻抗、放松)，配有语音指导、时长和强度提示，患者可跟随练习。 |
|  | 穿戴设备联动 | 蓝牙连接并获取指定的医疗器械(如血压计、血糖仪、穿戴设备)数据，自动上传至云端，无需手动输入。 |
|  | 实时预警 | 当手动录入或设备上传的数据超出医生设定的个人化安全范围时，APP进行温和但明确的弹窗提示，并建议患者休息或联系医生 |
|  | 紧急处置 | 1、支持异常中断检测(长时间无数据或骤停)<br>2、一键SOS(呼叫120/家属)<br>3、地理位置自动推送给家属或社区医生 |
|  | 遗漏警报通知 | 患者长时间未执行任务或未上报数据，系统会发送提醒通知；若连续遗漏关键任务(如服药、运动等)，可能触发对家属或社区端的通知 |
| 数据统计 | 居家生理参数 | 以折线图等形式直观展示患者关键生理参数(如晨起血压、静息心率)的历史变化趋势 |
|  | 康复训练记录统计 | 一键记录训练项目的实际完成情况(如运动时长、达到的心率、疲劳程度) |
| 沟通协同 | 消息中心 | 接收来自医院或社区医生的文字、语音或视频消息(如处方调整通知、鼓励话语)，也可向医生发送非紧急的图文咨询。 |
| 健康教育 | 健康社区 | 以短视频、图文卡片等轻松形式，接收个性化的心脏康复知识(如低盐饮食技巧、如何识别危险信号) |
|  | 社群(待定) | 在严格匿名和管理下，提供安全的病友交流圈，分享经验，互相鼓励 |

## 四、系统技术设计
### 1、数据安全
患者健康数据的加密传输(TLS 1.2+)、加密存储、访问控制(基于角色的权限管理)、数据匿名化处理(用于科研分析时)。

### 2、隐私保护
符合《个人信息保护法》和《GB/T 42062-2022 医疗器械 风险管理》要求，获取用户知情同意，明确数据使用范围。

### 3、网络安全
系统满足网络安全等级保护(至少二级) 要求。描述防攻击、防篡改、入侵检测、安全审计、数据备份与灾难恢复等措施。

### 4、设备接入安全
规范蓝牙、Wi-Fi等可穿戴设备接入的认证协议，防止非法设备接入或数据窃取。
